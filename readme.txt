Репозиторий всегда находится в папке.
Метаданные - инфа о репозитории.
У обычной папки (не препозитория) нет метаданных.

Реп хранит все файлы и папки проекта.
Коммит - фиксация изменений в репо.
Ветка - отдельная линия разработки.
Для разработки отдельной фичи создается отдельня ветка. 
Мерж - слияние веток. 
Форк - копия чужого репо, можно внести изменения и предложить их вледльцу репо.
Пул - внесение  изменений из одной ветки репо в другую. 

git init - инициализация репо - создение папки с метаданными и историей изменений.
git status - информация о текущем состоянии репозитория (что было изменено Мodified / удалено / добавлено Staged / Untracked)
git commit - добавление изменения в репозиторий. У каждого коммита есть ИД.
git shoow HEAD^ - посмотреть состояние коммита. 

С чего начать: 
1. Создать / открыть папку
2. Открыть терминал 
3. Убедиться, что находимся в нужной папке -> команд pwd 
    3.1 cd.. поднимаемся на дирректорию выше 
4. Вносим изменения.
5. git add имя файла - добавить в стейдж - очередь будущего коммита.
6. git commit -m "commit1" - добавлем изменени в файлах в ветку.
